TRACE Method Enabled

TRACE reflects the full HTTP request back to the client. In combination with browser vulnerabilities it enables Cross-Site Tracing (XST) — stealing cookies and headers.

🔍
Server Headers Disclose Version

Server: header returns the full stack — Apache 1.3.20, mod_ssl version, and OpenSSL version. Attackers use this to pinpoint exactly which exploits apply.

🔒 SSLv2 — Why It Matters

deprecated protocol
🔴
SSLv2 is cryptographically broken

Released 1995, deprecated 2011, prohibited by RFC 6176 in 2011. Vulnerable to DROWN attack — an attacker capturing SSLv2 traffic can decrypt modern TLS sessions on the same server.

🟣
Ciphers offered by this server

SSL2_RC4_128, SSL2_DES_64_CBC, SSL2_DES_192_EDE3_CBC, SSL2_RC2_128_CBC — all considered weak or broken by modern standards. Export-grade ciphers also present.

Key Findings — Information Disclosure
🔴

Server Headers — Full Stack Disclosure

HTTP response headers reveal Apache 1.3.20, mod_ssl 2.8.4, OpenSSL 0.9.6b, and Red Hat Linux. Attackers do not need to guess — the server announces exactly what is running and which CVEs apply.

🟣

Webalizer Stats Page Exposed

Webalizer web usage statistics are publicly accessible. This leaks internal traffic patterns, referrer data, and may expose directory structure or usernames in URL paths.

⚠️

Default Apache Test Page Visible

The default Red Hat Apache test page is showing on port 80 — no custom site deployed. This confirms a default installation with no hardening applied, increasing confidence that other defaults are also in place.

🔵

TRACE Method Enabled on HTTP

HTTP TRACE is enabled. Nmap flagged this as potentially risky. In combination with other client-side vulnerabilities, it can be used for Cross-Site Tracing (XST) to steal session tokens.

🔐

SSLv2 Enabled — DROWN Attack Surface

Port 443 accepts SSLv2 connections with broken cipher suites including export-grade RC4 and DES variants. SSLv2 is prohibited by RFC 6176. Presence on the same server as HTTPS creates DROWN attack conditions.
