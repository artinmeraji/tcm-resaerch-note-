WHAT IS SSH
// Protocol
SSH — Secure Shell — is a cryptographic network protocol used to operate network services securely over an unsecured network.

It replaced older plaintext protocols like Telnet and rsh that sent everything including passwords in cleartext over the wire.
// What you can do with it
→ Remote shell access to a machine
→ Execute commands on a remote host
→ Transfer files (scp / sftp)
→ Port forwarding / tunneling
→ Pivot through networks
→ Grab banners for reconnaissance
// Why we tried it here
Two reasons to probe SSH on a target even if you don't expect a login:

1. Banner grabbing — the SSH banner often leaks the exact OS, distro, and software version before you even authenticate. That intel narrows down exploits.

2. Version fingerprinting — old SSH versions (like OpenSSH 2.9p2 here) have their own known CVEs and support weak legacy algorithms that can be abused.

In this case: no banner was exposed. Dead end — but the attempt itself tells us important things about how old and misconfigured this system is.

// 03
FINAL COMMAND — FLAG BY FLAG
// every flag and what it unlocks
-oKexAlgorithms +diffie-hellman-group-exchange-sha1,
diffie-hellman-group1-sha1		 Re-enables old key exchange algorithms that modern OpenSSH disables. The + means "add to the default list" rather than replacing it.
// Without this: "no matching key exchange method found"
-oHostKeyAlgorithms +ssh-rsa Re-enables ssh-rsa as an accepted host key type. Modern OpenSSH deprecated it because SHA-1 is weak. The server identifies itself with an RSA key so we must allow it.
// Without this: "no matching host key type found"
-oCiphers +aes128-cbc Re-enables AES-128-CBC as an allowed cipher for encrypting the session. CBC mode ciphers are deprecated in modern SSH due to padding oracle attacks. Old servers only offer them.
// Without this: cipher negotiation would fail after handshake
-oPubkeyAcceptedKeyTypes +ssh-rsa Separately tells OpenSSH to accept ssh-rsa for public key authentication. HostKeyAlgorithms controls the server's identity key. This one controls what key types are accepted for user login.
// Both needed because they control different parts of the handshake
